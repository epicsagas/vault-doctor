# Security Policy

## Reporting a vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via [GitHub Security Advisories](https://github.com/epicsagas/vault-doctor/security/advisories/new).

Please include:

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Possible mitigation

## Response time

- Acknowledgment within 48 hours
- Initial assessment within 5 business days
- Patch release timeline depends on severity

## Scope

This policy covers the vault-doctor CLI tool itself. It does **not** cover:

- Third-party dependencies (report to the upstream maintainer)
- Issues in user vault files or configurations

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |
